1. Introduction 
1.1 Purpose 
This document provides a detailed description of the requirements for the Enterprise Ticket 
Management System (ETMS). The purpose of this system is to enable structured logging, tracking, 
assignment, approval, and resolution of service requests within an organization. 
This SRS serves as a reference for: 
• 
Business stakeholders 
• 
Project managers 
• 
Software developers 
• 
QA and testing teams 
• 
System administrators 
 
1.2 Scope 
The Enterprise Ticket Management System is a web-based application designed to: 
• 
Centralize issue and request management 
• 
Track ticket lifecycle from creation to closure 
• 
Enforce Service Level Agreements (SLA) 
• 
Implement configurable approval workflows 
• 
Provide dashboards and analytical reports 
• 
Maintain audit logs for compliance 
• 
Support role-based access control 
The system will be fully dynamic. All configurations such as roles, categories, priorities, SLA rules, and 
approval requirements will be managed through the administrative interface without requiring code 
modification. 
 
1.3 Definitions and Abbreviations 
SLA – Service Level Agreement 
RBAC – Role-Based Access Control 
API – Application Programming Interface 
JWT – JSON Web Token 
ETMS – Enterprise Ticket Management System

2. Overall Description 
2.1 Product Perspective 
The system follows a three-tier architecture: 
1. Presentation Layer – React-based web interface 
2. Application Layer – Django REST API 
3. Data Layer – MySQL relational database 
Communication between frontend and backend will occur via secure RESTful APIs over HTTPS using 
JSON data format. 
The system is designed to be scalable, modular, and cloud-ready. 
 
2.2 Product Functions 
The system shall provide: 
• 
Secure authentication and authorization 
• 
Ticket creation and lifecycle management 
• 
Ticket assignment and escalation 
• 
SLA calculation and monitoring 
• 
Approval workflows for selected ticket categories 
• 
Email and in-app notifications 
• 
Dashboard and reporting features 
• 
Audit logging and history tracking

3. System Features 
 
3.1 Authentication and Authorization 
Description: 
Provides secure access control. 
Functional Requirements: 
• 
Users shall authenticate using email and password. 
• 
System shall issue JWT tokens upon successful login. 
• 
Access shall be controlled using role-based permissions. 
• 
Passwords shall be securely hashed and stored. 
 
3.2 Ticket Management 
Description: 
Handles complete ticket lifecycle. 
Functional Requirements: 
• 
System shall allow creation of tickets. 
• 
Each ticket shall have a unique auto-generated ticket number. 
• 
Tickets shall include category, priority, and description. 
• 
Tickets shall follow defined status transitions: 
o 
Open 
o 
Assigned 
o 
In Progress 
o 
Waiting for User 
o 
Pending Third Party 
o 
Resolved 
o 
Closed 
• 
All changes shall be recorded in audit logs. 
 
3.3 SLA Management 
• 
SLA deadlines shall be calculated automatically. 
• 
System shall track SLA compliance. 
• 
Notifications shall be triggered before SLA breach. 
• 
SLA reports shall be generated. 
 
3.4 Approval Workflow 
• 
Certain ticket categories shall require approval. 
• 
System shall support approval and rejection. 
• 
Approval history shall be maintained. 
 
3.5 Notification System 
• 
System shall send email notifications. 
• 
System shall store in-app notifications. 
• 
Notifications shall trigger on: 
o 
Ticket creation 
o 
Status change 
o 
Assignment 
o 
Approval request 
o 
SLA breach 
 
3.6 Reporting and Dashboard 
• 
Dashboard shall display ticket summaries. 
• 
Reports shall include: 
o 
Open vs Closed tickets 
o 
SLA compliance 
o 
Agent performance 
• 
Reports shall be exportable (CSV/PDF).
